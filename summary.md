Prompt Instruksi Pengembangan Aplikasi: Personal Portfolio Website
1. Profil Pengguna & Tujuan:

Profil: Backend Developer.

Tujuan Utama: Personal branding dan memamerkan proyek teknis.

Karakteristik: Website harus menonjolkan logika backend, arsitektur sistem, dan efisiensi kode daripada sekadar estetika visual.

2. Proyek Utama Saat Ini (Initial Project):

Nama Proyek: OCR API (Optical Character Recognition).

Highlight Teknis: Dokumentasi API (Swagger/Postman), Diagram Arsitektur, Metrik Performa (akurasi/kecepatan), dan Code Snippet.

3. Kebutuhan Fitur (Functional Requirements):

Scalable Project Gallery: Sistem grid/list yang mudah ditambah proyek baru di masa depan tanpa merombak layout.

Deep-Dive Project Page: Halaman detail khusus proyek yang mencakup:

Deskripsi masalah dan solusi.

Tech stack tags.

Diagram arsitektur sistem.

Link repositori (GitHub) dan Live Demo/Sandbox.

Hero Section: Bio ringkas sebagai Backend Developer dan tombol CTA (Download CV/Kontak).

Skills & Contact: Daftar keahlian teknis dan integrasi media sosial profesional (LinkedIn/GitHub).

4. Kebutuhan Teknis (Non-Functional Requirements):

Tech Stack Recommended: Framework yang mendukung SEO dan performa tinggi (seperti Next.js, Astro, atau Hugo).

Data Management: Gunakan pendekatan berbasis data (JSON, Markdown, atau Headless CMS) agar penambahan proyek di masa depan tidak memerlukan koding ulang (cukup tambah data).

Performance: Skor Google Lighthouse tinggi (>90) dan responsif di perangkat mobile/desktop.

5. Tugas Model AI:

Buatlah struktur folder dan skema data (JSON/Markdown) untuk portfolio ini.

Rancang boilerplate kodenya dengan fokus pada kecepatan loading dan kebersihan kode.

Implementasikan layout yang memungkinkan detail proyek backend terlihat profesional (terutama untuk proyek OCR API).
